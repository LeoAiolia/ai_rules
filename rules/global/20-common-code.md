# 通用代码规范

## 命名

- 禁止无意义命名（`a` / `b` / `tmp` / `data1` / `info2`）。
- 命名必须语义自解释，优先于写注释。
- 布尔变量前缀：`is` / `has` / `should` / `can`。
- 常量、类、函数遵循各语言主流约定（Dart/Swift/Kotlin 的 lowerCamelCase / UpperCamelCase 等）。

## 魔法值

- 禁止裸数字、裸字符串作为业务含义出现在代码中（包括硬编码颜色、尺寸、超时时长、状态码）。
- 必须通过 **枚举 / `const` / 配置对象 / 设计 Token** 定义并集中管理。
- 例外：`0` / `1` / `-1` 等无业务含义的边界值；单元测试中的固定输入。

## 函数设计

| 约束 | 要求 |
|---|---|
| 单一职责 | 一个函数只做一件事 |
| 最大长度 | ≤ 50 行（不含注释和空行） |
| 最大嵌套 | ≤ 3 层 |
| 超限处理 | 拆分为多个私有小函数 |

## 枚举处理（强制）

- 枚举判断**必须使用 `switch`**，禁止 `if/else` 链。
- 必须穷举所有 case；无法穷举时必须提供 `default`，且 `default` 中要有明确兜底（日志 / 异常 / 安全默认值），禁止空分支。

## DRY 与可扩展

- 禁止重复代码；≥2 处出现的相同逻辑必须抽取。
- 禁止写死分支逻辑，优先 **枚举驱动 / 策略模式**。
- 所有外部依赖必须支持注入（DI），保证可测试。

## 异常处理

- 所有异步操作必须处理错误，禁止空 `catch`。
- 必须有**兜底分支** + **日志记录**；不允许吞掉异常后无任何痕迹。
- 关键路径异常须包含：**输入参数（脱敏）**、**当前状态**、**错误栈**。

## 架构分层（禁止跨层调用）

```
UI 层  →  业务层 (ViewModel / Controller)  →  数据层 (Repository)  →  网络层 (ApiClient)
```

- UI 层**禁止**直接调用 API / 写 HTTP / 写业务逻辑。
- 所有数据必须通过 **Repository** 获取。
- 所有 API 必须走**统一封装层**，统一处理异常、超时、错误码。
- 响应数据必须映射为 Model，禁止把原始 JSON / Map 透传到上层。

## 日志

- 关键路径必打日志：入参（脱敏）、出参、耗时、错误。
- 日志分级：`debug` / `info` / `warn` / `error`，按重要性使用，禁止把所有日志打成 `info`。
- 禁止打印敏感信息（Token、密码、手机号 / 身份证 / 银行卡明文、用户精确位置）。脱敏示例：`138****5678`。
- 生产环境默认关闭 `debug` 级别。
