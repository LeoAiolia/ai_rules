# 输出前自检清单（通用）

AI 在输出代码前必须逐项核对：

**命名 / 魔法值**

- [ ] 是否存在无意义命名（`a` / `tmp` / `data1`）
- [ ] 布尔变量是否带 `is` / `has` / `should` / `can` 前缀
- [ ] 是否存在魔法值（含硬编码颜色、尺寸、超时、状态码）

**函数 / 结构**

- [ ] 函数是否 ≤50 行、≤3 层嵌套、单一职责
- [ ] 是否存在 ≥2 处重复逻辑未抽取（DRY）
- [ ] 是否依赖具体实现而非接口（DI 是否就位）

**控制流**

- [ ] 枚举判断是否使用 `switch` 且穷举/兜底
- [ ] 所有异步操作是否处理错误（无空 `catch`）
- [ ] 异常路径是否有日志（含脱敏入参 / 状态 / 错误栈）

**架构 / 日志**

- [ ] 是否违反**分层**（UI 直连网络 / UI 直连 API / 跨层调用）
- [ ] 响应数据是否映射为 Model（禁止 Map / 原始 JSON 透传）
- [ ] 日志分级是否合理（不要全打 `info`）；是否打印了敏感信息

**其他**

- [ ] 是否使用了**已废弃 API**
- [ ] 不确定项是否标注 `// TODO: [需确认]`

平台专属自检见对应平台规则文件末尾（仅当项目 `AGENTS.md` 注入了平台规则时生效）。

**违反任一项必须先修正再输出。**
